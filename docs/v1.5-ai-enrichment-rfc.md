# Dokument projektowy v1.5 — Wzbogacanie opisu AI

**Stan:** SZKIC — wymaga decyzji Arka przed implementacją.
**Źródło:** przeniesienie wzorca z mobile K16 (`project_inwentaryzacja_mobile.md`) + ponowne użycie logiki z `~/Projekty_software/muzeum-inwentarz/inwentarz.py`.
**Branch:** `feature/v1.5-ai-enrichment`.

## Problem

Desktop ma `description TEXT` w `eksponaty` ale opisy są **wpisywane ręcznie** przez Arka w MainWindow. Często krótkie ("Amiga 1200"), bez kontekstu (epoka, charakterystyka, wartość historyczna).

Mobilna apka już ma identyfikację AI (Claude Opus generuje pełny `Artefakt` ze zdjęcia). Desktop NIE — tylko edytuje istniejące rekordy z bazy bez AI.

**Cel:** w PreviewDialog (renderuje markdown) dodać guzik "🪄 Wzbogać opis AI" który:
1. Pobierze zdjęcia eksponatu z bazy (BLOB w `photos`)
2. Wyśle do Anthropic Claude wraz z istniejącym `description` + danymi (nazwa, producent, model) jako kontekst
3. Pokaże nowy opis w oknie podglądu (stary obok nowego, oba jako markdown)
4. Arek akceptuje → UPDATE description w bazie

## Decyzje do podjęcia (PROSZĘ ZATWIERDZIĆ)

### 1. Architektura — A vs B vs C

**Opcja A: Bezpośrednio HTTP z C++ do Anthropic API** ⭐ rekomendowana
- Plus: samodzielnie, bez zależności od Pythona, bez dodatkowych kroków instalacji
- Plus: wzorce z mobile `ApiClient.cpp` (prepareRequest, formatNetworkError)
- Minus: ~250 linii nowego kodu (HTTP + parsowanie JSON + powtarzalna definicja schematu Anthropic w C++)

**Opcja B: C++ wywołuje `inwentarz.py` przez QProcess**
- Plus: ponowne użycie istniejącego pipeline'u w Pythonie 1:1
- Minus: Python jako zależność desktopa (instalator Windows = Python wbudowany), `pip install`, venv setup
- Minus: koszmar pakowania na różne platformy (.deb OK, .dmg + .exe trudne)

**Opcja C: C++ wysyła REST do mobilnego backendu FastAPI**
- Plus: jedno źródło prawdy, cienki klient w desktopie
- Minus: wymaga deploymentu backendu na hosting (K14 z mobilnej apki — nie skończony)
- Minus: desktop zależny od dostępności backendu

### 2. Gdzie trzymamy klucz API

- Tylko zmienna środowiskowa `ANTHROPIC_API_KEY` (programiści OK, zwykły użytkownik niewygodnie)
- Tylko QSettings (rejestr/ini plaintext — kompromis bezpieczeństwa, ale dla apki desktopowej 1-osobowej OK)
- **Mieszane:** zmienna środowiskowa wygrywa, w razie braku QSettings z prośbą o wpisanie + ostrzeżeniem o przechowywaniu lokalnie

### 3. Potwierdzanie kosztu

- Ostrzeżenie kosztu przed wysłaniem ("wyślę X zdjęć do AI, koszt ~$0.05, kontynuować?")
- Albo bezpośrednio bez pytania (szybciej, ale ryzyko niekontrolowanych $$)

### 4. Model

- **Claude Opus 4.7** (najlepsza jakość, ~$5/1M wejście / $25/1M wyjście, vision dobre)
- Claude Sonnet 4.6 (3× tańszy, dobry dla większości eksponatów)
- Konfigurowalny w Ustawieniach?

### 5. Strategia wejścia

- **Wzbogacanie** — wyślij istniejący opis + dane + zdjęcia, polecenie "wzbogać"
- **Zastąpienie** — zignoruj istniejący, generuj od zera
- Arek wybiera per wywołanie?

### 6. Akcja zapisu

- Auto-zapis do bazy po Akceptacji w oknie podglądu
- Albo tylko podgląd — Arek wraca do MainWindow do ręcznego zapisu (większa kontrola)

## Proponowana architektura (zakładając A + mieszany klucz + Opus + wzbogacanie + auto-zapis po potwierdzeniu)

### Nowa klasa `AiEnrichmentService` (analogicznie do `DatabaseBackupService`)

```cpp
class AiEnrichmentService : public QObject {
    Q_OBJECT
public:
    explicit AiEnrichmentService(QObject *parent = nullptr);

    /// Asynchronicznie — emit `enrichmentReady(newDescription)` lub `enrichmentError(msg)`.
    void enrichDescription(const QString &existingDescription,
                          const QList<QByteArray> &photoBlobs,
                          const QString &itemName,
                          const QString &itemVendor,
                          const QString &itemModel);

signals:
    void enrichmentReady(const QString &newDescription);
    void enrichmentError(const QString &message);

private:
    QString readApiKey() const;  // zmienna środowiskowa → QSettings → prośba
    QNetworkAccessManager *m_nam;
};
```

### Modyfikacje PreviewDialog

- Nowy guzik "🪄 Wzbogać opis AI" w pasku guzików (obok "Edytuj")

## Co może pójść nie tak (analiza wstępna)

- **Niekontrolowane koszty:** Arek klika guzik 100× zamiast 5× → $5 spalone. Zabezpieczenie: okno potwierdzenia z kosztem.
- **Wyciek klucza API:** Arek zapomni że klucz w QSettings (plaintext). Zabezpieczenie: ostrzeżenie + opcja zmiennej środowiskowej.
- **Zmiana schematu API:** Anthropic zmienia format odpowiedzi → break. Zabezpieczenie: walidacja schematu w stylu Pydantic w C++ (ręczne sprawdzenia JSON).
- **Limit zapytań:** Arek przetwarza wiele eksponatów naraz → 429. Zabezpieczenie: obsługa nagłówka retry-after.
- **Niedostępność modelu:** Opus 4.7 wycofany w przyszłości → rezerwowo Sonnet.
- **Rozmiar BLOB-ów zdjęć:** 200MB zdjęć w 1 rekordzie → request za duży. Zabezpieczenie: limit max 5 zdjęć lub zmniejszenie do 1024px przed wysłaniem.
- **Wstrzyknięcie markdown:** AI generuje markdown z linkami które otwiera QTextBrowser → potencjalny XSS desktopowy. Zabezpieczenie: `setOpenExternalLinks(false)` (już jest) + sanityzacja markdown przed renderem.
