Plik {\ttfamily \mbox{\hyperlink{photoitem_8cpp}{photoitem.\+cpp}}} zawiera implementację metod klasy {\ttfamily \mbox{\hyperlink{class_photo_item}{Photo\+Item}}}, która reprezentuje interaktywną miniaturę zdjęcia w scenie graficznej Qt ({\ttfamily QGraphics\+View}) w aplikacji inwentaryzacyjnej. Klasa rozszerza {\ttfamily QGraphics\+Pixmap\+Item} o obsługę zdarzeń myszy (kliknięcia, podwójne kliknięcia) i hover (najechanie, opuszczenie), a także wizualne zaznaczanie za pomocą czerwonej ramki. Dzięki dziedziczeniu po {\ttfamily QObject}, umożliwia emisję sygnałów ({\ttfamily clicked}, {\ttfamily double\+Clicked}, {\ttfamily hovered}, {\ttfamily unhovered}) do integracji z logiką aplikacji, np. z klasą {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}} dla wyboru lub podglądu zdjęć eksponatów.\hypertarget{md_documentation_photoitem_cpp_autotoc_md102}{}\doxysection{Struktura kodu}\label{md_documentation_photoitem_cpp_autotoc_md102}

\begin{DoxyEnumerate}
\item {\bfseries{Inicjalizacja}}\+:
\begin{DoxyItemize}
\item Konstruktor tworzy obiekt jako {\ttfamily QObject} i {\ttfamily QGraphics\+Pixmap\+Item}, ustawia flagi {\ttfamily m\+\_\+pressed} i {\ttfamily m\+\_\+selected} na {\ttfamily false}, tworzy ramkę ({\ttfamily m\+\_\+frame}) jako dziecko, włącza zdarzenia hover ({\ttfamily set\+Accept\+Hover\+Events(true)}), wyłącza zdarzenia dotykowe i ustawia ramkę jako niewidoczną ({\ttfamily Qt\+::\+No\+Pen}, {\ttfamily Qt\+::\+No\+Brush}).
\item Ramka ma wyższy priorytet rysowania ({\ttfamily set\+ZValue(1)}), aby była widoczna nad miniaturą.
\end{DoxyItemize}
\item {\bfseries{Interfejs graficzny}}\+:
\begin{DoxyItemize}
\item Wyświetla miniaturę zdjęcia (np. 80x80 pikseli w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}) w {\ttfamily QGraphics\+View}.
\item Przy zaznaczeniu ({\ttfamily m\+\_\+selected == true}), pokazuje czerwoną ramkę o szerokości 2 piksele, obejmującą miniaturę z marginesem (-\/2, +2 piksele).
\end{DoxyItemize}
\item {\bfseries{Interakcje}}\+:
\begin{DoxyItemize}
\item {\bfseries{Kliknięcia}} ({\ttfamily mouse\+Press\+Event})\+:
\begin{DoxyItemize}
\item Prawe kliknięcie emituje {\ttfamily clicked} (np. dla podglądu zdjęcia w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}).
\item Lewe kliknięcie jest przekazywane do {\ttfamily QGraphics\+Pixmap\+Item}, aby umożliwić wykrycie podwójnego kliknięcia.
\item Inne przyciski są obsługiwane domyślnie.
\end{DoxyItemize}
\item {\bfseries{Podwójne kliknięcia}} ({\ttfamily mouse\+Double\+Click\+Event})\+:
\begin{DoxyItemize}
\item Lewe podwójne kliknięcie emituje {\ttfamily double\+Clicked} (np. dla pełnoekranowego podglądu).
\item Inne przyciski są obsługiwane domyślnie.
\end{DoxyItemize}
\item {\bfseries{Hover}}\+:
\begin{DoxyItemize}
\item Najechanie ({\ttfamily hover\+Enter\+Event}) emituje {\ttfamily hovered} (np. dla podświetlenia, nieużywane w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}).
\item Opuszczenie ({\ttfamily hover\+Leave\+Event}) emituje {\ttfamily unhovered} (np. dla usunięcia podświetlenia).
\end{DoxyItemize}
\item Wszystkie zdarzenia są przekazywane do {\ttfamily QGraphics\+Pixmap\+Item} dla domyślnej obsługi.
\end{DoxyItemize}
\item {\bfseries{Zaznaczenie}}\+:
\begin{DoxyItemize}
\item {\ttfamily set\+Selected} ustawia {\ttfamily m\+\_\+selected} i wywołuje {\ttfamily update\+Frame}.
\item {\ttfamily update\+Frame} rysuje czerwoną ramkę (2 piksele) dla {\ttfamily m\+\_\+selected == true} lub ukrywa ją ({\ttfamily Qt\+::\+No\+Pen}) dla {\ttfamily m\+\_\+selected == false}.
\item Ramka jest dopasowana do granic miniatury ({\ttfamily bounding\+Rect}) z marginesem.
\end{DoxyItemize}
\item {\bfseries{Integracja}}\+:
\begin{DoxyItemize}
\item Sygnał {\ttfamily clicked} jest podłączony w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}} do {\ttfamily on\+Photo\+Clicked} dla wyboru miniatury (np. do usunięcia lub podglądu).
\item Sygnał {\ttfamily double\+Clicked} może być użyty do pełnoekranowego podglądu (niezrealizowane w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}).
\item Sygnały {\ttfamily hovered} i {\ttfamily unhovered} są zdefiniowane, ale nieużywane w obecnej implementacji.
\end{DoxyItemize}

\item Kod nie został zmodyfikowany, zgodnie z wymaganiami użytkownika.
\item Klasa jest zoptymalizowana dla miniatur zdjęć w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}, ale sygnały {\ttfamily hovered} i {\ttfamily unhovered} nie są wykorzystywane, co sugeruje możliwość dodania podświetlania lub tooltipów.
\item Flaga {\ttfamily m\+\_\+pressed} jest zdefiniowana, ale nieużywana, co może wskazywać na planowane funkcje (np. przeciąganie miniatur).
\item Ramka zaznaczenia jest statyczna (czerwona, 2 piksele), ale może być dostosowana (np. kolor, grubość) w przyszłych wersjach.
\item Obsługa prawego przycisku dla {\ttfamily clicked} może być rozszerzona o menu kontekstowe (np. \char`\"{}\+Usuń\char`\"{}, \char`\"{}\+Podgląd\char`\"{}).
\item Klasa nie zależy od bazy danych ani innych komponentów aplikacji, co czyni ją uniwersalną dla scen graficznych Qt.
