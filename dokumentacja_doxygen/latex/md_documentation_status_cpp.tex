Plik {\ttfamily \mbox{\hyperlink{status_8cpp}{status.\+cpp}}} zawiera implementację metod klasy {\ttfamily status}, odpowiedzialnej za zarządzanie statusami eksponatów (np. \char`\"{}\+Nowy\char`\"{}, \char`\"{}\+Używany\char`\"{}, \char`\"{}\+Uszkodzony\char`\"{}) w aplikacji inwentaryzacyjnej. Klasa umożliwia dodawanie, edytowanie i usuwanie statusów w tabeli {\ttfamily statuses} w bazie danych My\+SQL, współpracując z interfejsem użytkownika ({\ttfamily QList\+View}, {\ttfamily QLine\+Edit}) oraz klasą {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}} w celu odświeżania combo boxów statusów po zapisaniu zmian.\hypertarget{md_documentation_status_cpp_autotoc_md116}{}\doxysection{Struktura kodu}\label{md_documentation_status_cpp_autotoc_md116}

\begin{DoxyEnumerate}
\item {\bfseries{Inicjalizacja}}\+:
\begin{DoxyItemize}
\item Konstruktor ustawia interfejs ({\ttfamily ui-\/\texorpdfstring{$>$}{>}setup\+Ui}), pobiera połączenie z bazą My\+SQL ({\ttfamily default\+\_\+connection}), ustala tytuł okna, podłącza sloty dla przycisków (Dodaj, Edytuj, Usuń, OK, Anuluj) i wywołuje {\ttfamily refresh\+List} do wyświetlenia statusów.
\item Inicjalizuje {\ttfamily m\+\_\+main\+Window} jako {\ttfamily nullptr}.
\end{DoxyItemize}
\item {\bfseries{Interfejs użytkownika}}\+:
\begin{DoxyItemize}
\item Zawiera {\ttfamily QList\+View} do wyświetlania statusów (sortowanych alfabetycznie), {\ttfamily QLine\+Edit} do wprowadzania nazwy statusu oraz przyciski do zarządzania statusami i zamykania okna.
\item {\ttfamily QSql\+Query\+Model} dostarcza dane do {\ttfamily QList\+View}.
\end{DoxyItemize}
\item {\bfseries{Zarządzanie statusami}}\+:
\begin{DoxyItemize}
\item {\bfseries{Dodawanie}} ({\ttfamily on\+Add\+Clicked})\+:
\begin{DoxyItemize}
\item Pobiera nazwę z {\ttfamily QLine\+Edit}, waliduje (niepusta).
\item Generuje UUID dla statusu.
\item Wykonuje {\ttfamily INSERT} do tabeli {\ttfamily statuses}, odświeża listę i czyści pole.
\item {\bfseries{Błąd}}\+: Linia {\ttfamily QUuid\+::create\+Uuid().to\+String(\+QUuid\+::\+Without\+Braces);} generuje UUID, ale nie przypisuje go (wynik jest odrzucany).
\end{DoxyItemize}
\item {\bfseries{Edycja}} ({\ttfamily on\+Edit\+Clicked})\+:
\begin{DoxyItemize}
\item Sprawdza, czy wybrano status w {\ttfamily QList\+View}.
\item Otwiera {\ttfamily QInput\+Dialog} z aktualną nazwą, waliduje nową nazwę.
\item Pobiera ID statusu po nazwie, wykonuje {\ttfamily UPDATE} i odświeża listę.
\end{DoxyItemize}
\item {\bfseries{Usuwanie}} ({\ttfamily on\+Delete\+Clicked})\+:
\begin{DoxyItemize}
\item Sprawdza, czy wybrano status.
\item Prosi o potwierdzenie w {\ttfamily QMessage\+Box}.
\item Wykonuje {\ttfamily DELETE} dla nazwy statusu i odświeża listę.
\end{DoxyItemize}
\item Wyświetla komunikaty o błędach SQL lub brakujących wyborach.
\end{DoxyItemize}
\item {\bfseries{Integracja z {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}}}\+:
\begin{DoxyItemize}
\item {\ttfamily on\+Ok\+Clicked} wywołuje {\ttfamily load\+Combo\+Box\+Data} na combo boxie statusów w {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}} (jeśli ustawione) i zamyka okno z wynikiem {\ttfamily accept()}.
\item {\ttfamily set\+Main\+Window} przechowuje referencję do {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}} dla odświeżania.
\end{DoxyItemize}

\item Usuwanie i edycja statusów opierają się na nazwie, co może powodować problemy przy duplikatach nazw (brak unikalności w schemacie bazy).
\item Połączenie z bazą danych ({\ttfamily default\+\_\+connection}) jest współdzielone z innymi komponentami (np. {\ttfamily \mbox{\hyperlink{class_main_window}{Main\+Window}}}, {\ttfamily models}).
